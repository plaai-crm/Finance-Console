# ตั้งค่าแจ้งเตือนรายจ่ายผ่าน LINE (สำหรับ Owner)

ระบบจะส่งสรุป **รายจ่ายที่ครบกำหนดภายใน 3 วันและยังไม่ได้จ่าย** เข้า LINE ให้ Owner
**อัตโนมัติทุกเช้า 08:00 น.** โดยไม่ต้องเปิดคอมหรือแอปค้างไว้ (ทำงานบน Supabase)

> ต้องเชื่อม Supabase ให้เสร็จก่อน (ดูไฟล์ `README.md` ขั้นตอน 1–3) แล้วจึงทำตามนี้
> ทำครั้งเดียวจบ ใช้เวลาประมาณ 15 นาที

---

## ส่วนที่ 1 — สร้าง LINE Official Account (บัญชี LINE ของธุรกิจ)

ระบบส่งข้อความผ่าน “LINE Official Account (OA)” ซึ่งใครก็ตามที่ **เพิ่มเป็นเพื่อน** จะได้รับแจ้งเตือน
(ให้เฉพาะ Owner เพิ่มเป็นเพื่อน ก็จะเห็นแค่ Owner)

1. เข้า **https://manager.line.biz** แล้วล็อกอินด้วยบัญชี LINE ของ Owner
2. กด **Create** / **สร้างบัญชีใหม่** ตั้งชื่อ เช่น `แจ้งเตือนการเงิน` แล้วสร้างให้เสร็จ
3. ใช้มือถือของ Owner เปิดแอป LINE → สแกน QR หรือค้นหาบัญชี OA นี้ แล้วกด **เพิ่มเพื่อน**
   (ใครที่อยากรับแจ้งเตือนด้วย ก็ให้เพิ่มเพื่อนบัญชีนี้)

## ส่วนที่ 2 — เปิด Messaging API แล้วคัดลอก Token

1. ใน LINE Official Account Manager เข้า **Settings (ตั้งค่า)** → **Messaging API**
2. กด **Enable / Use Messaging API** (ถ้ายังไม่เปิด) — ระบบจะให้ผูกกับ **LINE Developers** ทำตามที่จอบอกจนเสร็จ
3. ไปที่ **https://developers.line.biz/console** → เลือก Provider → เลือก Channel ของ OA นี้
4. แท็บ **Messaging API** เลื่อนลงหา **Channel access token (long-lived)** กด **Issue** แล้ว **คัดลอก** ข้อความยาว ๆ เก็บไว้
   (นี่คือค่าที่ต้องใช้ในขั้นตอนถัดไป — เก็บเป็นความลับ อย่าเผยแพร่)

## ส่วนที่ 3 — ติดตั้งการแจ้งเตือนบน Supabase

1. เปิดไฟล์ **`line_notify.sql`** ด้วย Notepad/TextEdit
2. หาบรรทัดที่เขียนว่า `<<< แก้ตรงนี้` — แทนข้อความ `ใส่_CHANNEL_ACCESS_TOKEN_ของคุณตรงนี้`
   ด้วย Token ที่คัดลอกมาจากส่วนที่ 2 (วางไว้ **ระหว่างเครื่องหมาย ' ' เดิม**) เช่น:

   ```sql
   line_token text := 'AbCdEf1234....(token ยาว ๆ)....xyz';
   ```

3. **บันทึกไฟล์** แล้วคัดลอกเนื้อหาทั้งหมด
4. เข้า Supabase → **SQL Editor** → **New query** → วาง → กด **Run**
   - ถ้าขึ้น error เรื่อง `pg_cron` หรือ `pg_net` ให้ไปที่เมนู **Database → Extensions**
     ค้นหาและกดเปิด **pg_cron** และ **pg_net** แล้วรัน SQL ใหม่อีกครั้ง

## ส่วนที่ 4 — ทดสอบว่าเข้า LINE จริง

1. ในแอป Finance Console ลองเพิ่ม **รายจ่าย** สักรายการ ตั้งวันที่เป็น **วันนี้หรือพรุ่งนี้** และ **ยังไม่ติ๊กว่าจ่าย**
2. กลับไปที่ Supabase SQL Editor พิมพ์คำสั่งนี้แล้วกด Run:

   ```sql
   select fn_line_expense_reminder();
   ```

3. ภายในไม่กี่วินาที Owner ควรได้รับข้อความสรุปใน LINE 🎉
   - ถ้าไม่ได้รับ: ตรวจว่า Owner กด “เพิ่มเพื่อน” OA แล้ว และ Token ถูกต้อง

## สรุปการทำงาน

- ทำงานอัตโนมัติทุกวัน **08:00 น. เวลาไทย**
- ส่งเฉพาะเมื่อ **มีรายจ่ายค้างจ่ายที่ครบกำหนดภายใน 3 วัน** (ถ้าไม่มี จะไม่ส่ง ไม่รบกวน)
- พอ Owner หรือผู้บันทึกติ๊กว่า “จ่ายแล้ว” ในแอป รายการนั้นจะไม่ถูกแจ้งอีก

### ปรับแต่งเพิ่มเติม (ถ้าต้องการ)
- **เปลี่ยนเวลา**: ในไฟล์ `line_notify.sql` ค่า `'0 1 * * *'` คือ 01:00 UTC (= 08:00 ไทย)
  อยากได้ 07:00 ไทย ใช้ `'0 0 * * *'` · อยากได้ 20:00 ไทย ใช้ `'0 13 * * *'`
- **หยุดแจ้งเตือน**: รัน `select cron.unschedule('line_expense_reminder');`

> หมายเหตุ: บริการเดิม “LINE Notify” ถูกปิดไปแล้ว ระบบนี้จึงใช้ **LINE Messaging API** ซึ่งเป็นวิธีที่ LINE รองรับอย่างเป็นทางการ
